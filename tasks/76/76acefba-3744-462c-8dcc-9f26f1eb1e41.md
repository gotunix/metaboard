# Implementation Plan: t-1 (Implement color-coded status indicators)

## Context
The current dashboard uses plain text for statuses. We want to use `lipgloss` to add visual distinction between different states (ACTIVE, BACKLOG, etc.).

## Technical Approach
- Update `internal/ui/styles.go` with color definitions.
- Modify `StatusStyle` function to return appropriate colors.
- Integrate these styles into the `RenderDashboard` and `RenderBacklog` loops.

## Acceptance Criteria
- [x] ACTIVE shows as GREEN.
- [ ] BACKLOG shows as CYAN.
- [ ] COMPLETED shows as MAGENTA.
- [ ] CANCELLED shows as GRAY.
