# Implementation Plan: t-4 (Add 'metaboard doctor' command)

## Context
Users need a way to verify the integrity of their metadata (e.g., broken links, missing files).

## Technical Approach
- Create `cmd/metaboard/doctor.go`.
- Implement checks for:
  - Orphaned tasks/stories.
  - Broken parent/child links.
  - Invalid JSON syntax in data files.

## Acceptance Criteria
- [ ] Command runs via `./metaboard doctor`.
- [ ] Reports all found issues with suggestions for fixing.
